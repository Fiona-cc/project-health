# doc-rules.md — 文档侧检查规则（C2 断链 / C3 超大文档）

> audit v1。确定性检查。**C2 的第一原则：宁漏勿错**——误报一个断链，用户就不再信任整个工具。

---

## 扫描哪些文档

- 仓库根：`CLAUDE.md`、`AGENTS.md`、`README.md`
- `docs/**/*.md`
- **若项目中存在任意 `SKILL.md`**（即这是个含 skill 的项目），额外扫描：
  - `**/SKILL.md`
  - `**/references/**/*.md`
  - 目的：让 project-health 能**检查自己**的 `audit/SKILL.md` 与 `audit/references/*.md`。
- （其它 `*.md` 本版不主动扫，避免噪声。）

---

## C2 · 断链（路径 + 命令）

对每篇文档，逐行提取"引用"，判断目标是否存在。**路径引用与命令引用对代码块的处理不同**（见下）。

### 一、路径引用（保守提取，**跳过代码块**）

**预处理**：markdown 链接提取（下述第 1 类）**先剔除 fenced code block（一对 ` ``` ` 之间）和 inline 代码（单反引号 `` ` `` 之间）的内容**——因为 `[x](y)` 一旦写在代码块或反引号里，渲染后只是**字面文本、不是真链接**（文档讲解链接语法时常这么写，如 `` `[文字](目标)` ``、`` `[file:line](file#Lline)` ``）。跳过它们能消掉最常见的一类误报。反引号里的**真实路径**由下述第 2 类负责，不受影响。

**提取以下两类，其余一律不提取**：
1. **Markdown 链接** `[文字](目标)` 中，`目标`为本地相对路径。
   - 排除：`http://`、`https://`、`mailto:`、以 `#` 开头的纯锚点。
   - 若目标含 `#anchor`，只取 `#` 前的路径部分。
2. **反引号 token** 且**明确像仓库路径**——满足任一：
   - 以 `./` 或 `../` 开头；或
   - **第一段是仓库中真实存在的顶层目录**（先 `ls` 仓库根拿到顶层目录集合，token 的第一段命中才算路径）。
3. **不提取**：无路径的裸文件名（如正文里的 `service.py`）、明显是示例/占位的路径（如 `path/to/xxx`、`your-project/`）。

**存在性判定（markdown 语义，与 scanner 一致）**：
- markdown 链接：普通相对路径**严格相对该文档目录**解析；`/开头`→仓库根；**不再额外尝试仓库根 fallback**（fallback 会掩盖"`docs/x.md` 引用 `src/a.ts` 应指向 `docs/src/a.ts`=断链"这类错误）。
- 反引号路径：保守规则，相对**仓库根**。
- `../` 解析越出项目根 → 判为断链；任何 `scheme:`（http/mailto/tel/data/file…）与 `//host` → 跳过（非本地路径）。
（目标以 `/` 结尾视为目录，判目录存在。）

**输出每条**：`<文档>:<行号> → <引用目标>` + "指向不存在的路径"。

### 二、命令引用（**扫描代码块 + 反引号**，仅 npm 系，且有 package.json 时）

> 与路径检查相反：命令**要**扫 fenced code block 和反引号——因为 README 里的运行命令常写在代码块里（如 ```` ```bash\nnpm run build\n``` ````）。

- 若仓库根**无 `package.json`**（或它损坏/非对象）→ 整项跳过命令检查（`skipped_checks`: `invalid_package_json`），不误报一片。
- 在**代码块内和反引号内**只提取 `npm run <script>` / `pnpm run <script>` / `yarn run <script>`。
  **v1 不查简写**（`yarn build` / `pnpm build`）——它们易和 `install` / `add` / `exec` 等内置命令混淆。
- `<script>` 不在 `package.json` 的 `scripts` 键中 → 🔴。
- mvn/gradle/make 等其它命令体系：**本版不检查**。

**输出每条**：`<文档>:<行号> → <命令>` + "引用了不存在的脚本"。

### C2 防误报红线
- 顶层目录集合必须来自**实际 `ls`**，不是猜。
- 拿不准的一律**不报**。宁可漏一个真断链，也不冤枉一个正常词。

---

## C3 · 超大文档

- **对象**：与 C2 相同的文档集（根三件 + `docs/**/*.md`；若项目含 skill，另加 `**/SKILL.md` 与 `**/references/**/*.md`）。
- **度量**：总行数（含空行即可，文档空行本身也占篇幅）。
- **判级**（config `thresholds.doc_warn`，默认 500）：
  - 行数 ≥ 500 → ⚠️。
- **输出每条**：`<文档>:<行数>` + "文档过长，建议拆分/压缩；已被迭代淘汰的旧记录可归档为一句话摘要"。
